## Endpoint

```http
GET /v3/adapters/order-get-mcd-v1/orders/{orderId}
```

## Autenticación

Envía el API key como bearer token.

```http
Authorization: Bearer <API_KEY>
```

El API key debe tener permisos `admin`, pertenecer a la misma marca del pedido y tener acceso al local del pedido.

## Comportamiento

- La llamada obtiene una orden ecommerce de Justo y la devuelve en el formato de evento `orderNew` solicitado por McDonald's.
- Los identificadores de orden se envían como string.
- `app_id` se envía vacío porque Justo no lo tiene como dato propio de la orden.
- `app_shop_id` se envía como el `storeId` de Justo.
- Los productos y opciones usan el ID original McDonald's guardado en `metadata.id` por `menu-put-mcd-v1`. Si no existe, Justo usa el `externalId` sin el prefijo `{storeId}:`.
- Los montos se devuelven con la misma unidad entera que expone Justo para la orden.

## Ejemplo

```bash
curl -X GET \
  "https://api.service.getjusto.com/v3/adapters/order-get-mcd-v1/orders/{orderId}" \
  -H "Authorization: Bearer <API_KEY>"
```

## Mapeo inicial

| Campo MCD | Fuente Justo |
| --- | --- |
| `app_id` | Vacío. Justo no lo tiene como dato de la orden. |
| `app_shop_id` | `order.storeId` |
| `data.order_id` | `order._id` como string |
| `order_info.order_id` | `order._id` como string |
| `order_info.order_index` | `order.code` convertido a número cuando aplica |
| `order_info.expected_arrived_eta` | `order.deliverAt` en Unix timestamp segundos |
| `order_info.expected_cook_eta` | `order.createdAt + expectedPreparationDuration` |
| `order_info.price.order_price` | `order.totalPrice` |
| `order_info.price.items_discount` | `order.totalDiscount` |
| `order_info.price.others_fees.service_price` | `order.serviceFee` |
| `order_info.price.store_charged_delivery_price` | `order.deliveryFee` |
| `order_items[].app_item_id` | `product.metadata.id` o `product.externalId` sin `{storeId}:` |
| `order_items[].sub_item_list[].app_item_id` | `modifierOption.metadata.id` o `modifierOption.externalId` sin `{storeId}:` |

## Códigos pendientes por confirmar

Este adapter incluye un mapeo inicial para poder validar el formato con McDonald's, pero los códigos numéricos deben confirmarse con MDC antes de considerarlos definitivos.

| Campo | Mapeo inicial |
| --- | --- |
| `status` | `pending = 100`, `waiting = 200`, `preparing = 300`, `delivering = 400`, `done = 600`, `cancelled = 900` |
| `pay_type` / `pay_method` | `cash = 2`, otros medios de pago = `1` |
| `pay_channel` | `cash = 0`, otros medios de pago = `150` |
| `delivery_type` | `delivery = 1`, `go = 2`, `serve = 3` |
| `promo_type` | Descuentos de producto u orden = `2` |
| `c_cancel_preference` | `7` |
| `fulfillment_mode` | `0` |
