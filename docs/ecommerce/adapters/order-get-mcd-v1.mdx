</Warning>

## Endpoint

```http
GET /v3/adapters/order-get-mcd-v1/orders/{orderId}
```

## Autenticación

Envía el API key como bearer token.

```http
Authorization: Bearer <API_KEY>
```

El API key debe tener permisos `admin`, pertenecer a la misma marca del pedido y tener acceso al local del pedido.

## Comportamiento

- McDonald's debe recibir los webhooks ecommerce estándar de Justo para enterarse de nuevos pedidos y cambios de estado.
- Después de recibir un webhook, McDonald's puede llamar este adapter con el `orderId` del webhook para consultar la orden en formato `orderNew`.
- La llamada obtiene una orden ecommerce de Justo y la devuelve en el formato de evento `orderNew` solicitado por McDonald's.
- El ETA que Flex Digital necesita para pasar la orden a cocina corresponde a `order_info.expected_arrived_eta`: fecha estimada de llegada del repartidor al local, en Unix timestamp segundos.
- Justo solo envía `expected_arrived_eta` cuando existe un despacho con repartidor y se puede calcular la llegada al local. Si no hay repartidor asignado o no hay ETA calculable, se envía `null`.
- Los identificadores de orden se envían como string.
- `app_id` se envía vacío porque Justo no lo tiene como dato propio de la orden.
- `app_shop_id` se envía como el `storeId` de Justo.
- Los productos y opciones usan el ID original McDonald's guardado en `metadata.id` por `menu-put-mcd-v1`. Si no existe, Justo usa el `externalId` sin el prefijo `{storeId}:`.
- Los montos se devuelven con la misma unidad entera que expone Justo para la orden.
- Los campos numéricos no monetarios que dependen de un dato no informado por Justo se envían como `null`, no como `0`. En campos de precios, descuentos, cobros o devoluciones, `0` significa cero.

## Ejemplo

```bash
curl -X GET \
  "https://api.service.getjusto.com/v3/adapters/order-get-mcd-v1/orders/{orderId}" \
  -H "Authorization: Bearer <API_KEY>"
```

## Campos de respuesta

### Evento

| Campo | Significado |
| --- | --- |
| `app_id` | Identificador de aplicación McDonald's. Justo lo envía vacío. |
| `app_shop_id` | Identificador del local que recibe la orden. |
| `type` | Tipo de evento. Para este adapter siempre es `orderNew`. |
| `timestamp` | Fecha de creación de la orden en Unix timestamp segundos. |
| `data.order_id` | Identificador único de la orden. |

### Información de la orden

| Campo | Significado |
| --- | --- |
| `order_info.order_id` | Identificador único de la orden. |
| `order_info.status` | Estado actual de la orden. Ver valores en la sección de códigos. |
| `order_info.order_index` | Número visible de la orden. |
| `order_info.order_channel` | Canal de venta. Para este adapter se envía `Food`. |
| `order_info.remark` | Observación general de la orden. Si no hay observación, se envía vacío. |
| `order_info.country` | País de la orden en código ISO alfa-2, por ejemplo `PE`. |
| `order_info.city_id` | Identificador de la ciudad del local. |
| `order_info.timezone` | Zona horaria usada para interpretar la operación del local. |
| `order_info.pay_type` | Tipo de pago. Ver valores en la sección de códigos. |
| `order_info.pay_method` | Método de pago. Usa los mismos valores que `pay_type`. |
| `order_info.pay_channel` | Canal de pago. Ver valores en la sección de códigos. |
| `order_info.delivery_type` | Tipo de entrega. Ver valores en la sección de códigos. |
| `order_info.delivery_eta` | ETA general de entrega. Este adapter no lo usa y lo envía como `null`. |
| `order_info.expected_cook_eta` | ETA esperado de preparación. Este adapter no lo calcula y lo envía como `null`. |
| `order_info.expected_arrived_eta` | Fecha estimada de llegada del repartidor al local, en Unix timestamp segundos. Si no hay repartidor asignado o no hay ETA calculable, se envía `null`. |
| `order_info.create_time` | Fecha de creación de la orden en Unix timestamp segundos. |
| `order_info.pay_time` | Fecha de pago en Unix timestamp segundos. Si la orden aún no está pagada, se envía `null`. |
| `order_info.complete_time` | Fecha de finalización de la orden. Si no está disponible, se envía `null`. |
| `order_info.cancel_time` | Fecha de cancelación de la orden. Si no está disponible, se envía `null`. |
| `order_info.shop_confirm_time` | Fecha de confirmación del local. Si no está disponible, se envía `null`. |
| `order_info.c_cancel_preference` | Código fijo de preferencia de cancelación. Ver valores en la sección de códigos. |
| `order_info.fulfillment_mode` | Modo de fulfillment. Ver valores en la sección de códigos. |

### Precios

Todos los campos de precios, descuentos, cobros y devoluciones son numéricos. Si no hay monto asociado, se envía `0`; `null` no se usa para montos.

| Campo | Significado |
| --- | --- |
| `order_info.price.order_price` | Total de productos de la orden, sin costo de delivery. |
| `order_info.price.items_discount` | Descuento total aplicado a productos. Si no hay descuento, se envía `0`. |
| `order_info.price.delivery_discount` | Descuento aplicado al delivery. Si no hay descuento, se envía `0`. |
| `order_info.price.others_fees.service_price` | Cargo de servicio. Si no hay cargo de servicio, se envía `0`. |
| `order_info.price.shop_paid_money` | Monto pagado por el local. Si no aplica, se envía `0`. |
| `order_info.price.refund_price` | Monto devuelto. Si no hay devolución, se envía `0`. |
| `order_info.price.store_charged_delivery_price` | Monto de delivery cobrado al local. Si no aplica, se envía `0`. |

### Local

| Campo | Significado |
| --- | --- |
| `order_info.shop.shop_id` | Identificador del local. |
| `order_info.shop.app_shop_id` | Identificador del local para la integración. |
| `order_info.shop.shop_addr` | Dirección del local. |
| `order_info.shop.shop_name` | Nombre del local. |
| `order_info.shop.shop_phone` | Lista de teléfonos del local. Si no hay teléfono, se envía una lista vacía. |
| `order_info.shop.shop_phone[].calling_code` | Código telefónico del país. Si no está disponible, se envía `null`. |
| `order_info.shop.shop_phone[].phone` | Número de teléfono del local. |
| `order_info.shop.shop_phone[].type` | Tipo de teléfono. Para este adapter se envía `1`. |

### Dirección de entrega

| Campo | Significado |
| --- | --- |
| `order_info.receive_address.uid` | Identificador del cliente. Si no está disponible, se envía vacío. |
| `order_info.receive_address.name` | Nombre completo del cliente. |
| `order_info.receive_address.first_name` | Primer nombre del cliente. |
| `order_info.receive_address.last_name` | Apellido o resto del nombre del cliente. |
| `order_info.receive_address.calling_code` | Código telefónico del país. Si no está disponible, se envía vacío. |
| `order_info.receive_address.phone` | Teléfono del cliente. |
| `order_info.receive_address.city` | Ciudad o zona de la dirección. |
| `order_info.receive_address.country_code` | País de la dirección en código ISO alfa-2. |
| `order_info.receive_address.poi_address` | Dirección principal de entrega. |
| `order_info.receive_address.house_number` | Complemento de dirección, departamento o referencia. |
| `order_info.receive_address.poi_lat` | Latitud de la dirección. Si no está disponible, se envía `null`. |
| `order_info.receive_address.poi_lng` | Longitud de la dirección. Si no está disponible, se envía `null`. |
| `order_info.receive_address.coordinate_type` | Sistema de coordenadas. Para este adapter se envía `wgs84`. |
| `order_info.receive_address.poi_display_name` | Nombre visible de la dirección. |

| Campo | Significado |

| `order_info.order_items[].sub_item_list[].sub_item_list` | Lista anidada de opciones. Para este adapter se envía vacía. |
| `order_info.order_items[].sub_item_list[].sales_type` | Tipo de venta de la opción. Para este adapter se envía `unit`. |

### Promociones

| Campo | Significado |
| --- | --- |
| `order_info.promotions` | Promociones aplicadas a nivel de orden. Si no hay promociones, se envía una lista vacía. |
| `order_info.order_items[].promo_type` | Tipo de promoción aplicada al producto. Solo se envía si hay descuento. |
| `order_info.order_items[].promotion_detail` | Detalle de la promoción aplicada al producto. Solo se envía si hay descuento. |
| `order_info.order_items[].promo_list` | Lista de promociones aplicadas al producto. Solo se envía si hay descuento. |
| `promo_type` | Tipo de promoción. Ver valores en la sección de códigos. |
| `promo_discount` | Monto descontado. |
| `shop_subside_price` | Monto subsidiado por el local. |

## Valores codificados

| Campo | Valores |
| --- | --- |
| `status` | `100`: pendiente, `200`: esperando confirmación, `300`: preparando, `400`: en despacho, `600`: completada, `900`: cancelada. |
| `pay_type` / `pay_method` | `1`: pago no efectivo, `2`: efectivo. |
| `pay_channel` | `0`: efectivo, `150`: pago no efectivo. |
| `delivery_type` | `1`: delivery, `2`: retiro, `3`: consumo en local. |
| `promo_type` | `2`: descuento aplicado a producto u orden. |
| `c_cancel_preference` | `7`: valor fijo enviado por el adapter. |
| `fulfillment_mode` | `0`: valor fijo enviado por el adapter. |
