---
openapi: post /ecommerce.order.delivery.updated
---

Este webhook se envía cuando cambia el estado del despacho asociado a un pedido. Permite detectar asignaciones de driver, cambios de ETA ligados a la transición de estado y casos en que el despacho vuelve a un estado anterior.

## ETA de llegada al local

Cuando el despacho está en estado `active` y Justo tiene la ubicación actual del driver, el payload incluye `delivery.estimatedArrivalAtStoreAt`.

Este campo indica la fecha estimada de llegada del driver al local. Se calcula usando la distancia en línea recta entre la ubicación del driver y el punto de retiro (`fromLocation`), asumiendo una velocidad de 15 km/h.

```json
{
  "type": "ecommerce.order.delivery.updated",
  "data": {
    "delivery": {
      "status": "active",
      "estimatedArrivalAtStoreAt": "2026-05-16T20:12:00.000Z",
      "deliveryExpectedAt": "2026-05-16T20:45:00.000Z"
    }
  }
}
```

Si el despacho no está `active` o todavía no hay ubicación del driver, `estimatedArrivalAtStoreAt` puede venir vacío.
