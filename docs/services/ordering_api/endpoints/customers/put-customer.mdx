---

Crea o actualiza el customer propio de la app autenticada.

Todas las llamadas usan `Authorization: Bearer <token>` con scope `orderingApi`.

## Endpoint

```http
PUT /v3/ordering/customers/{externalCustomerId}
```

## Path parameters

<ParamField name="externalCustomerId" type="string" required>
  ID del customer en el sistema externo. Es independiente por app.
</ParamField>

## Body

<ParamField name="email" type="string">
  Email real del customer, usado para notificaciones.
</ParamField>

<ParamField name="phone" type="string">
  Teléfono real del customer.
</ParamField>

<ParamField name="firstName" type="string">
  Nombre del customer.
</ParamField>

<ParamField name="lastName" type="string">
  Apellido del customer.
</ParamField>

## Response

<ResponseField name="success" type="boolean" required>
  Indica si la llamada fue procesada correctamente.
</ResponseField>

<ResponseField name="data" type="OrderingCustomer" required>
  Customer creado o actualizado.
</ResponseField>

## Data object

### OrderingCustomer

Customer propio de la app Ordering API.

<ResponseField name="externalCustomerId" type="string">
  ID del customer en el sistema externo.
</ResponseField>

<ResponseField name="email" type="string | null">
  Email real del customer, usado para notificaciones.
</ResponseField>

<ResponseField name="phone" type="string | null">
  Teléfono real del customer.
</ResponseField>

<ResponseField name="firstName" type="string | null">
  Nombre del customer.
</ResponseField>

<ResponseField name="lastName" type="string | null">
  Apellido del customer.
</ResponseField>

<ResponseField name="createdAt" type="string">
  Fecha de creación del customer.
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Fecha de última actualización del customer.
</ResponseField>
