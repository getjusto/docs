Obtiene billing guardado en checkout-state para customer y website.

Todas las llamadas usan `Authorization: Bearer <token>` con scope `orderingApi`.

## Endpoint

```http
GET /v3/ordering/customers/{externalCustomerId}/websites/{domain}/billing
```

## Path parameters

<ParamField name="externalCustomerId" type="string" required>
  ID del customer en el sistema externo. Es independiente por app.
</ParamField>

<ParamField name="domain" type="string" required>
  Dominio canónico del website Justo, sin protocolo ni `www.`.
</ParamField>

## Response

<ResponseField name="success" type="boolean" required>
  Indica si la llamada fue procesada correctamente.
</ResponseField>

<ResponseField name="data" type="OrderingBilling | null" required>
  Datos de facturación guardados, o null si no existen.
</ResponseField>

## Data object

### OrderingBilling

Datos de facturación guardados para el checkout.

<ResponseField name="name" type="string | null">
  Nombre o razón social.
</ResponseField>

<ResponseField name="document" type="string | null">
  Documento tributario.
</ResponseField>

<ResponseField name="email" type="string | null">
  Email de facturación.
</ResponseField>

<ResponseField name="address" type="string | null">
  Dirección de facturación.
</ResponseField>

<ResponseField name="activityType" type="string | null">
  Giro o actividad económica.
</ResponseField>
