Lista medios de pago disponibles para el local y tipo de entrega.

Todas las llamadas usan `Authorization: Bearer <token>` con scope `orderingApi`.

## Endpoint

```http
GET /v3/ordering/customers/{externalCustomerId}/websites/{domain}/cart/payment-methods
```

## Path parameters

<ParamField name="externalCustomerId" type="string" required>
  ID del customer en el sistema externo. Es independiente por app.
</ParamField>

<ParamField name="domain" type="string" required>
  Dominio canónico del website Justo, sin protocolo ni `www.`.
</ParamField>

## Response

<ResponseField name="success" type="boolean" required>
  Indica si la llamada fue procesada correctamente.
</ResponseField>

<ResponseField name="data" type="OrderingPaymentMethod[]" required>
  Medios de pago disponibles.
</ResponseField>

## Data object

### OrderingPaymentMethod

Medio de pago disponible.

<ResponseField name="paymentMethod" type="string">
  Código del medio de pago.
</ResponseField>

<ResponseField name="label" type="string">
  Nombre visible del medio de pago.
</ResponseField>

<ResponseField name="canDeliver" type="boolean">
  Indica si permite despacho.
</ResponseField>

<ResponseField name="isIntegration" type="boolean">
  Indica si el pago depende de una integración externa.
</ResponseField>

<ResponseField name="allowedDeliveryTypes" type="delivery | go | serve[]">
  Tipos de entrega permitidos para este medio de pago.
</ResponseField>

<ResponseField name="requiresRoundTrip" type="boolean">
  Indica si requiere viaje ida y vuelta.
</ResponseField>

<ResponseField name="isAvailableInV3" type="boolean">
  Indica si está disponible para API v3.
</ResponseField>

<ResponseField name="isSecure" type="boolean">
  Indica si requiere validaciones seguras adicionales.
</ResponseField>
